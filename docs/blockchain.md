# 区块链基础知识

##  区块链是什么？一句话理解

区块链是一种**不可篡改的分布式数据库**，所有交易按顺序被打包进“区块”中，每个区块都链接前一个，像链条一样串起来。


##  数据结构

```
[区块0] ← [区块1] ← [区块2] ← [区块3] ← ...
```

每个区块就像一个“账本页”，写满交易记录后锁住，生成下一个页时会引用上一页的哈希值，从而保证全链一致和不可篡改。


##  关键概念解释

###  1. 区块（Block）

区块是区块链的最小“记录单位”，像账本中的一页。

| 字段           | 说明                                 |
| -------------- | ------------------------------------ |
| 区块高度       | 当前是链上的第几个区块（从 0 开始）   |
| 区块时间戳     | 被打包的时间                         |
| 上一个区块哈希 | 当前区块链接的上一个区块标识         |
| 交易列表       | 本区块内的所有交易记录               |
| Merkle Root    | 所有交易的哈希根，用于快速验证       |
| Gas 使用量     | 执行合约代码所消耗的计算资源         |

>  区块就像每秒新增的“数据快照”，被全网共同记账。


###  2. 交易（Transaction）

交易是用户在链上发起的操作请求，是写入区块的数据单位，包括：

- 转账：ETH、USDT、NFT 等
- 调用合约函数：如 mint、swap、stake、vote
- 批准授权：approve 合约控制你的 Token

每笔交易都包含签名、地址、gas、data 等字段，被节点验证后打包进区块。


###  3. 哈希（Hash）

哈希是把任意数据转成一串固定长度的加密“指纹”。

以太坊常用 Keccak256 算法（类似 SHA3），例如：

```
keccak256("hello world") = 0x47173285...d05f
```

**用途：**

- 校验数据是否被篡改（数据一致性）
- 建立 Merkle Tree 的结构
- 标识交易、区块、地址、合约等内容（不可逆）

**特点：**  
同输入 → 同输出；输入改一点，输出完全不同；无法反推出原文。

###  4. 节点（Node）

节点是运行区块链客户端的软件，可以同步、验证、广播区块和交易。

| 节点类型 | 功能                                         |
| -------- | -------------------------------------------- |
| 全节点   | 保留所有区块数据，可参与共识验证             |
| 轻节点   | 只下载区块头，用于快速验证                   |
| RPC 节点 | DApp 用于连接和读写链上数据的接口节点        |

举例：Alchemy、Infura、QuickNode、Chainstack 都是提供 RPC 节点服务的平台。


###  5. Gas（燃料）

执行交易和智能合约时所需的计算资源费用。

每笔交易必须设置 Gas Limit 和 Gas Price，用于激励区块生产者优先打包。

```
总手续费 = gasUsed × gasPrice（单位 Gwei）
```


##  ✅ 小结

| 概念   | 一句话总结                           |
| ------ | ------------------------------------ |
| 区块   | 区块链上的“账本页”，打包交易数据     |
| 交易   | 用户发起的操作，是状态变更的核心     |
| 哈希   | 不可逆的数字指纹，用于唯一标识       |
| 节点   | 链的服务器，负责同步验证数据         |
| Gas    | 支付矿工 / 验证者的运算燃料费用      |
